# Enterprise Composite Layouts

**Goal**: Prevent interface fragmentation by rigidly adhering to standardized, recognizable screen architectures for both Web and Mobile.

## Web / Desktop Layout Catalog

1. **LMR (Left - Middle - Right)**: Headers, Topbars, Toolbars.
2. **Sidebar - Content**: Layout divider (e.g., Talent Pool beside Office).
3. **Grid-Board**: Kanban, Metrics grid.
4. **Dashboard Bento**: Dashboards.
5. **Master-Detail**: Task review, Inbox.
6. **Holy Grail (App Shell)**: Primary Application Framework.
7. **Chat / Terminal Session**: Operator Console.
8. **Card Grid with Filters**: Marketplaces.
9. **Timeline / Gantt**: Schedules.
10. **Wizard / Stepper**: Onboarding flows.
11. **Settings / Properties Panel**: Detail inspector.
12. **Multi-Project Pipeline**: Vertical queuing lists.
13. **SAFe Topology / Org Chart**: Tree graphs, nested boxes.
14. **Agent Metrics Dashboard**: 2x2 density grid for single entity health.
15. **Multi-Monitor Terminal Mosaic**: 2x2 or 3x3 grids of live feeds.
16. **Activity Log / Event Stream**: Full screen vertical timeline.
17. **Project Batch Setup / Import**: Split layout: 50% Dropzone, 50% Review Table.
18. **Agent Pool Health Pulse**: Ultra high-density list with inline sparklines/pings.
19. **Resource Allocation Conflict Matrix**: Cross-tabular assignment grid.
20. **Role Template Builder Canvas**: Drag-and-drop workspace (Palette + Canvas).
21. **Alert Resolution Inbox**: Urgent escalation queues with embedded terminal and action bar.

## Mobile App Layout Catalog

22. **Bottom Tab Navigation**: Standard iOS/Android 3-5 tab root navigation.
23. **Stack Page (with Back Button/Swipe)**: Deep-dive view. Header contains back chevron on left, title in middle.
24. **Bottom Sheet Modal**: Contextual tasks preventing context switch. Drags up from the bottom (takes 50%-90% of screen height).
25. **Swipeable Feed / ViewPager**: Infinite vertical feed (TikTok style) or horizontal swiping between full-screen data cards.
26. **Floating Action Button (FAB) Stack**: Fixed button at bottom-right containing primary creative action (e.g., "Spawn Agent").

## iPad / Tablet Layout Catalog

27. **Adaptive Split View**: Fluid layouts that respond to `Compact` vs `Regular` width classes. Converts to Slide Over mobile layout in narrow context.
28. **Sidebar Navigation**: Dedicated left sidebar for Enterprise iPad apps, abandoning mobile-style bottom tabs.
29. **Pointer & Touch Hybrid**: Support Hover states for pointer morphing, while rigidly maintaining minimum 44pt touch targets for fingers.

For detailed descriptions on any of these, consult the compiled guides.
